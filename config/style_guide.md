# 우리말 번역·작성 스타일 가이드 (IR 보고서용)

출처 원칙: 『이렇게 해야 바로 쓴다』(한겨레신문사, 1994). 영어 실적·컨콜을
우리말로 옮길 때 **번역체를 버리고 자연스러운 우리말**로 쓰기 위한 규칙이다.
보고서를 쓰는 모델은 아래를 따른다. (HBM·DRAM·EPS·ASP 등 굳어진 기술·재무
용어는 그대로 쓰되, 그 밖의 문장은 우리말답게.)

## 1. 단어
- **'의' 남용 금지.** 명사+의+명사 사슬을 풀어 동사·구체 서술로. 예) `시민의 권리` → `시민이 지닌 권리`, `법원의 보수적 분위기` → `법원의 최근 보수적 분위기`.
- **'것' 줄이기.** 특히 `~할 것이다`(당위·강조)는 `~해야 한다`로. 예) `계획해야 할 것이다` → `계획해야 한다`.
- **명사화 풀기.** `~음/~기 + 하다/이다` 대신 동사로. 예) `근절이 되지 않았다` → `근절되지 않았다`, `~함을 시인했다` → `~했다고 시인했다`, `건설 중에 있다` → `건설하고 있다`.
- **불필요한 '-들' 빼기.** 복수가 문맥으로 분명하면 생략. 예) `많은 사람들이` → `많은 사람이`.
- **쉬운 말로.** 예) `언급을 회피했다` → `말하려 하지 않았다`, `~에 대해 언급` → `~을 말함`.
- **주관·과장어 자제.** `결코, 과연, 설마, 어차피, 심지어, 사상 최대` 같은 말과 최상급은 근거 있을 때만. 근거 없는 단정은 피한다.
- **외래어·번역투 글버릇 최소화.** 기술/재무 표준어가 아니면 우리말로. 예) `내러티브` → `서사/메시지`, `리스크` → `위험`(문맥에 따라), `포지셔닝` → `자리매김`(필요 시 병기).

## 2. 문장
- **짧게, 한 문장 한 생각.** 길고 복잡한 문장은 끊는다.
- **이중부정·에두름 금지.** 예) `다름 아니었다` → `~였다`, `~지 않을 수 없다` → `~해야 한다`.
- **명사문(`-이다`)을 동사·형용사문으로.** 예) `도망은 죽음이다` → `도망가면 죽는다`.
- **번역체 버리기.**
  - `have` 직역 `갖고 있다` → `있다`. 예) `아이 셋을 갖고 있다` → `아이가 셋 있다`.
  - **무생물 주어(물주구문) 지양**, 사람·현상을 주체로. 예) `거센 바람이 집을 흔들었다` → `거센 바람에 집이 흔들렸다`.
  - 영어식 강조 `It is ~ that`(= `~한 것은 ~이다`) 풀어 쓰기.
- **수식어는 꾸밈받는 말 바로 앞에.** 긴 관형절은 한 문장으로 독립.
- **주어·목적어·서술어 호응.** 복잡하면 빠진 주어·목적어를 다시 적는다.
- **어휘 중복 금지.** 예) `투고하여 주세요` → `보내 주세요`, `너무너무/굉장히` 남발 금지.
- **문장 끝을 명사·말줄임표로 닫지 않기.** 끝까지 서술한다(개조식 표·불릿은 예외로 명사형 허용).

## 3. 보고서 적용
- 표·불릿은 간결한 개조식(명사형) 허용. **본문·해석·시사점 문장은 위 규칙을 지켜 완결된 우리말 문장**으로.
- 영문 원문 발췌는 그대로 인용하고, **해석은 직역투가 아니라 뜻이 통하는 우리말**로 옮긴다.
- 톤은 객관·평서체("~한다/~이다"). 존칭·청유·감탄·의문은 쓰지 않는다.
